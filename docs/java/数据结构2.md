# Java 面试问题 - 算法与数据结构

## 1. 什么是时间复杂度和空间复杂度？

**时间复杂度**是指算法执行时间随输入规模增长的变化趋势，用于衡量算法的运行效率。通常使用大 O 符号（O-notation）来表示，它描述了在最坏情况下算法的时间增长速率。

**常见的时间复杂度**：

- O(1)：常数时间复杂度，算法执行时间不随输入规模变化

- O(log n)：对数时间复杂度，算法执行时间随输入规模呈对数增长

- O(n)：线性时间复杂度，算法执行时间随输入规模线性增长

- O(n log n)：线性对数时间复杂度，常见于高效的排序算法

- O(n²)：平方时间复杂度，常见于简单的排序算法

- O(n³)：立方时间复杂度，常见于矩阵乘法等算法

- O(2ⁿ)：指数时间复杂度，常见于递归算法

- O(n!)：阶乘时间复杂度，常见于旅行商问题等 NP 难问题

**空间复杂度**是指算法执行过程中所需的额外空间随输入规模增长的变化趋势，用于衡量算法的内存消耗。同样使用大 O 符号表示。

**常见的空间复杂度**：

- O(1)：常数空间复杂度，算法所需额外空间不随输入规模变化

- O(n)：线性空间复杂度，算法所需额外空间随输入规模线性增长

- O(n²)：平方空间复杂度，算法所需额外空间随输入规模平方增长

**时间复杂度和空间复杂度的关系**：

- 时间和空间是算法的两个重要性能指标

- 通常情况下，时间复杂度和空间复杂度是相互 trade-off 的，减少时间消耗可能会增加空间消耗，反之亦然

- 算法设计时需要根据实际需求平衡时间和空间复杂度

## 2. 常用的排序算法有哪些？它们的时间复杂度和空间复杂度？

**排序算法**是将一组数据按照特定顺序排列的算法，是算法设计中的基础问题。

**常用的排序算法**：

| 排序算法     | 平均时间复杂度  | 最坏时间复杂度  | 最好时间复杂度  | 空间复杂度 | 稳定性 |
| ------------ | --------------- | --------------- | --------------- | ---------- | ------ |
| **冒泡排序** | O(n²)           | O(n²)           | O(n)            | O(1)       | 稳定   |
| **选择排序** | O(n²)           | O(n²)           | O(n²)           | O(1)       | 不稳定 |
| **插入排序** | O(n²)           | O(n²)           | O(n)            | O(1)       | 稳定   |
| **希尔排序** | O(n log n)      | O(n²)           | O(n log n)      | O(1)       | 不稳定 |
| **归并排序** | O(n log n)      | O(n log n)      | O(n log n)      | O(n)       | 稳定   |
| **快速排序** | O(n log n)      | O(n²)           | O(n log n)      | O(log n)   | 不稳定 |
| **堆排序**   | O(n log n)      | O(n log n)      | O(n log n)      | O(1)       | 不稳定 |
| **计数排序** | O(n + k)        | O(n + k)        | O(n + k)        | O(n + k)   | 稳定   |
| **桶排序**   | O(n + k)        | O(n²)           | O(n)            | O(n + k)   | 稳定   |
| **基数排序** | O(d \* (n + k)) | O(d \* (n + k)) | O(d \* (n + k)) | O(n + k)   | 稳定   |

**排序算法的分类**：

- **比较排序**：通过比较元素大小进行排序，如冒泡排序、选择排序、插入排序、希尔排序、归并排序、快速排序、堆排序

- **非比较排序**：不通过比较元素大小进行排序，如计数排序、桶排序、基数排序

**排序算法的选择**：

- 小规模数据：插入排序、冒泡排序

- 中等规模数据：希尔排序、快速排序

- 大规模数据：归并排序、堆排序、快速排序

- 元素范围较小：计数排序、桶排序、基数排序

- 要求稳定排序：归并排序、冒泡排序、插入排序、计数排序、桶排序、基数排序

## 3. 什么是二分查找？二分查找的条件是什么？

**二分查找**是一种在有序数组中查找特定元素的高效算法，它通过不断将查找区间减半来缩小查找范围，直到找到目标元素或确定目标元素不存在。

**二分查找的条件**：

- 数组必须是有序的

- 数组必须支持随机访问（如数组）

**二分查找的基本思想**：

1. 初始化左边界 left=0，右边界 right=n-1，其中 n 是数组长度
2. 当 left ≤ right 时，执行以下操作：

    - 计算中间位置 mid = (left + right) / 2

    - 如果 arr[mid] == target，返回 mid

    - 如果 arr[mid] < target，将 left = mid + 1

    - 如果 arr[mid] > target，将 right = mid - 1

3. 如果循环结束后仍未找到目标元素，返回-1

**二分查找的时间复杂度**：O(log n)
**二分查找的空间复杂度**：O(1)（迭代实现）或 O(log n)（递归实现）

**二分查找的变体**：

- 查找第一个等于目标值的元素

- 查找最后一个等于目标值的元素

- 查找第一个大于等于目标值的元素

- 查找最后一个小于等于目标值的元素

**Java 实现示例**：

```java
// 基本二分查找
public int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 避免整数溢出
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}

// 查找第一个等于目标值的元素
public int binarySearchFirst(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;
    int result = -1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            result = mid;
            right = mid - 1; // 继续向左查找
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return result;
}
```

## 4. 什么是链表？链表的类型有哪些？

**链表**是一种线性数据结构，它由一系列节点组成，每个节点包含数据域和指针域。数据域存储数据，指针域存储下一个节点的引用。

**链表的特点**：

- 不需要连续的内存空间

- 插入和删除操作效率高，时间复杂度为 O(1)

- 访问元素效率低，时间复杂度为 O(n)

- 空间复杂度为 O(n)

**链表的类型**：

1. **单链表**：每个节点只有一个指针，指向后继节点

    - 结构简单，插入和删除操作只需要修改一个指针

    - 只能从头节点开始遍历

2. **双链表**：每个节点有两个指针，分别指向前驱节点和后继节点

    - 可以双向遍历，访问前驱节点的时间复杂度为 O(1)

    - 插入和删除操作需要修改两个指针

3. **循环链表**：尾节点的指针指向头节点，形成一个环

    - 可以循环遍历，适合处理环形结构

    - 可以从任意节点开始遍历

4. **双向循环链表**：双链表的尾节点指针指向头节点，头节点的前驱指针指向尾节点

    - 结合了双链表和循环链表的优点

    - 可以双向循环遍历

**链表的应用场景**：

- 频繁进行插入和删除操作的场景

- 内存空间不足或需要动态分配内存的场景

- 实现栈、队列、哈希表等数据结构

- 实现 LRU 缓存淘汰算法

**Java 中的链表实现**：

- **LinkedList**：双向链表实现，实现了 List 和 Deque 接口

- **LinkedHashSet**：基于链表的哈希表实现

- **LinkedHashMap**：基于链表的哈希表实现，保持插入顺序或访问顺序

## 5. 什么是栈和队列？它们的区别？

**栈（Stack）**是一种后进先出（LIFO - Last In First Out）的数据结构，只允许在一端进行插入和删除操作。

**栈的基本操作**：

- push()：将元素压入栈顶

- pop()：移除并返回栈顶元素

- peek()：返回栈顶元素但不移除

- isEmpty()：判断栈是否为空

- size()：返回栈中元素的个数

**队列（Queue）**是一种先进先出（FIFO - First In First Out）的数据结构，只允许在一端插入元素，在另一端移除元素。

**队列的基本操作**：

- offer()：将元素加入队列尾部

- poll()：移除并返回队列头部元素

- peek()：返回队列头部元素但不移除

- isEmpty()：判断队列是否为空

- size()：返回队列中元素的个数

**栈和队列的区别**：

| 特性         | 栈                                       | 队列                             |
| ------------ | ---------------------------------------- | -------------------------------- |
| **数据结构** | 后进先出（LIFO）                         | 先进先出（FIFO）                 |
| **插入操作** | 只允许在栈顶插入（push）                 | 只允许在队尾插入（offer）        |
| **删除操作** | 只允许在栈顶删除（pop）                  | 只允许在队头删除（poll）         |
| **访问操作** | 只允许访问栈顶元素（peek）               | 只允许访问队头元素（peek）       |
| **应用场景** | 函数调用、表达式求值、括号匹配、撤销操作 | 任务调度、消息队列、广度优先搜索 |

**栈和队列的实现方式**：

- 基于数组实现：固定大小，效率高

- 基于链表实现：动态大小，灵活

**Java 中的栈和队列实现**：

- **Stack**：基于 Vector 的栈实现，线程安全

- **LinkedList**：实现了 Deque 接口，可以作为栈或队列使用

- **ArrayDeque**：基于数组的双端队列实现，效率高

- **PriorityQueue**：优先队列，元素按优先级排序

## 6. 什么是树？树的类型有哪些？

**树**是一种非线性数据结构，它由节点组成，节点之间通过边连接，形成层次结构。树中有一个特殊的节点称为根节点，其他节点都是根节点的子孙节点。

**树的基本术语**：

- **根节点**：树的顶部节点，没有父节点

- **父节点**：有子节点的节点

- **子节点**：有父节点的节点

- **叶子节点**：没有子节点的节点

- **深度**：从根节点到当前节点的边数

- **高度**：从当前节点到最远叶子节点的边数

- **度**：节点的子节点个数

- **层次**：根节点为第 1 层，依次向下递增

**树的类型**：

1. **二叉树**：每个节点最多有两个子节点（左子节点和右子节点）

    - 完全二叉树：除最后一层外，每一层都被完全填满，最后一层的节点从左到右排列

    - 满二叉树：每一层都被完全填满，所有叶子节点都在同一层

    - 平衡二叉树：任意节点的左右子树高度差不超过 1

    - 二叉搜索树：左子树的所有节点值小于根节点，右子树的所有节点值大于根节点

2. **二叉搜索树（BST）**：左子树的所有节点值小于根节点，右子树的所有节点值大于根节点

    - 查找、插入、删除操作的平均时间复杂度为 O(log n)

    - 最坏情况下退化为链表，时间复杂度为 O(n)

3. **平衡二叉搜索树**：

    - **AVL 树**：任意节点的左右子树高度差不超过 1

    - **红黑树**：通过颜色标记和旋转操作保持树的平衡

4. **B 树**：一种多路平衡搜索树，用于磁盘存储

    - 每个节点可以有多个子节点

    - 适合处理大规模数据，减少磁盘 I/O 操作

5. **B+树**：B 树的变体，所有叶子节点通过链表连接

    - 所有数据都存储在叶子节点中

    - 适合范围查询，常用于数据库索引

6. **哈夫曼树**：带权路径长度最小的二叉树，用于数据压缩

7. **堆**：一种特殊的完全二叉树，分为最大堆和最小堆

    - 最大堆：父节点值大于等于子节点值

    - 最小堆：父节点值小于等于子节点值

**树的应用场景**：

- 实现文件系统

- 实现数据库索引

- 实现编译器的语法树

- 实现路由算法

- 实现决策树算法

- 实现堆排序、优先队列

## 7. 什么是二叉树？二叉树的遍历方式有哪些？

**二叉树**是一种每个节点最多有两个子节点的树结构，这两个子节点分别称为左子节点和右子节点。

**二叉树的遍历方式**：

**1. 深度优先遍历（DFS）**：

- **前序遍历（Pre-order）**：根节点 → 左子树 → 右子树

- **中序遍历（In-order）**：左子树 → 根节点 → 右子树

- **后序遍历（Post-order）**：左子树 → 右子树 → 根节点

**2. 广度优先遍历（BFS）**：

- **层序遍历（Level-order）**：按层次从上到下，每层从左到右遍历

**二叉树遍历的实现方式**：

**递归实现**：

```java
// 前序遍历
public void preOrder(TreeNode root) {
    if (root == null) return;
    System.out.print(root.val + " ");
    preOrder(root.left);
    preOrder(root.right);
}

// 中序遍历
public void inOrder(TreeNode root) {
    if (root == null) return;
    inOrder(root.left);
    System.out.print(root.val + " ");
    inOrder(root.right);
}

// 后序遍历
public void postOrder(TreeNode root) {
    if (root == null) return;
    postOrder(root.left);
    postOrder(root.right);
    System.out.print(root.val + " ");
}
```

**迭代实现**：

```java
// 前序遍历
public List<Integer> preOrder(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;
    Stack<TreeNode> stack = new Stack<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);
        if (node.right != null) stack.push(node.right);
        if (node.left != null) stack.push(node.left);
    }
    return result;
}

// 中序遍历
public List<Integer> inOrder(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    Stack<TreeNode> stack = new Stack<>();
    TreeNode curr = root;
    while (curr != null || !stack.isEmpty()) {
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;
        }
        curr = stack.pop();
        result.add(curr.val);
        curr = curr.right;
    }
    return result;
}

// 层序遍历
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
```

**二叉树遍历的应用**：

- 前序遍历：复制二叉树、打印表达式

- 中序遍历：二叉搜索树的排序遍历

- 后序遍历：计算二叉树的高度、释放二叉树内存

- 层序遍历：计算二叉树的宽度、查找二叉树的最大深度

## 8. 什么是二叉搜索树？二叉搜索树的特点？

**二叉搜索树（Binary Search Tree，BST）**是一种特殊的二叉树，它满足以下性质：

- 左子树的所有节点值小于根节点值

- 右子树的所有节点值大于根节点值

- 左子树和右子树也都是二叉搜索树

**二叉搜索树的特点**：

- **有序性**：中序遍历可以得到有序序列

- **高效的查找**：平均时间复杂度为 O(log n)，最坏情况下为 O(n)（退化为链表）

- **高效的插入和删除**：平均时间复杂度为 O(log n)，最坏情况下为 O(n)

- **不需要额外的空间**：只需要存储节点和指针

**二叉搜索树的基本操作**：

1. **查找操作**：

    - 从根节点开始，比较目标值与当前节点值

    - 如果目标值等于当前节点值，返回当前节点

    - 如果目标值小于当前节点值，在左子树中查找

    - 如果目标值大于当前节点值，在右子树中查找

    - 如果遍历到叶子节点仍未找到，返回 null

2. **插入操作**：

    - 从根节点开始，比较插入值与当前节点值

    - 如果插入值小于当前节点值，递归插入到左子树

    - 如果插入值大于当前节点值，递归插入到右子树

    - 如果当前节点为 null，创建新节点并返回

3. **删除操作**：

    - 查找目标节点

    - 如果目标节点是叶子节点，直接删除

    - 如果目标节点只有一个子节点，用子节点替换目标节点

    - 如果目标节点有两个子节点，找到右子树中的最小节点（或左子树中的最大节点），替换目标节点，然后删除该最小节点（或最大节点）

**二叉搜索树的缺点**：

- 可能退化为链表，导致查找、插入和删除操作的时间复杂度变为 O(n)

- 不平衡的二叉搜索树性能较差

**平衡二叉搜索树**：

- **AVL 树**：通过旋转操作保持树的平衡，任意节点的左右子树高度差不超过 1

- **红黑树**：通过颜色标记和旋转操作保持树的平衡，具有较好的插入和删除性能

- **Splay 树**：通过将访问的节点移到根节点来保持树的平衡，适合频繁访问的场景

**Java 中的二叉搜索树实现**：

- **TreeSet**：基于红黑树实现的有序集合

- **TreeMap**：基于红黑树实现的有序映射

## 9. 什么是平衡二叉树？AVL 树和红黑树的区别？

**平衡二叉树**是一种特殊的二叉搜索树，它通过某种机制保持树的平衡，防止树退化为链表，从而保证查找、插入和删除操作的时间复杂度为 O(log n)。

**AVL 树**是最早的平衡二叉搜索树，它通过旋转操作保持树的平衡，任意节点的左右子树高度差（平衡因子）的绝对值不超过 1。

**红黑树**是一种自平衡的二叉搜索树，它通过颜色标记（红色或黑色）和旋转操作保持树的平衡，满足以下性质：

1. 每个节点要么是红色，要么是黑色
2. 根节点是黑色
3. 所有叶子节点（NIL 节点）是黑色
4. 如果一个节点是红色，那么它的两个子节点都是黑色
5. 从任意节点到其每个叶子节点的所有路径都包含相同数目的黑色节点

**AVL 树和红黑树的区别**：

| 特性               | AVL 树                                                  | 红黑树                                               |
| ------------------ | ------------------------------------------------------- | ---------------------------------------------------- |
| **平衡条件**       | 平衡因子的绝对值不超过 1                                | 满足红黑树的 5 条性质                                |
| **平衡度**         | 高度平衡，平衡度高                                      | 近似平衡，平衡度较低                                 |
| **旋转操作**       | 插入和删除操作可能需要多次旋转                          | 插入操作最多需要 2 次旋转，删除操作最多需要 3 次旋转 |
| **查找性能**       | 查找性能更优，因为树的高度更低                          | 查找性能略差，但差异不大                             |
| **插入和删除性能** | 插入和删除操作的开销较大，因为需要多次旋转              | 插入和删除操作的开销较小，因为旋转次数少             |
| **存储空间**       | 只需要存储平衡因子或高度                                | 需要存储节点颜色                                     |
| **应用场景**       | 适合频繁查找、插入和删除操作较少的场景                  | 适合频繁插入和删除操作的场景                         |
| **Java 中的实现**  | TreeSet 和 TreeMap 在早期版本中使用，现在已被红黑树替代 | TreeSet、TreeMap、HashMap（JDK 8+）使用              |

**AVL 树和红黑树的选择**：

- 如果查找操作远多于插入和删除操作，选择 AVL 树

- 如果插入和删除操作频繁，选择红黑树

- 大多数情况下，红黑树的综合性能更好，应用更广泛

## 10. 什么是堆？堆的应用场景？

**堆**是一种特殊的完全二叉树，它满足以下性质：

- **最大堆**：父节点值大于等于子节点值

- **最小堆**：父节点值小于等于子节点值

**堆的特点**：

- 完全二叉树，适合用数组存储

- 堆顶元素是最大或最小元素

- 插入和删除操作的时间复杂度为 O(log n)

- 构建堆的时间复杂度为 O(n)

**堆的存储方式**：

- 使用数组存储堆，根节点存储在索引 1 的位置（便于计算）

- 对于索引为 i 的节点：

    - 父节点索引：i / 2

    - 左子节点索引：2 \* i

    - 右子节点索引：2 \* i + 1

**堆的基本操作**：

1. **插入操作**：

    - 将新元素插入到数组末尾

    - 向上调整（Heapify Up）：将新元素与父节点比较，如果不满足堆的性质，交换它们，直到满足堆的性质

2. **删除堆顶元素**：

    - 将堆顶元素与数组末尾元素交换

    - 删除数组末尾元素

    - 向下调整（Heapify Down）：将新的堆顶元素与子节点比较，如果不满足堆的性质，与较大（或较小）的子节点交换，直到满足堆的性质

3. **构建堆**：

    - 从最后一个非叶子节点开始，依次向下调整每个节点

    - 时间复杂度为 O(n)

**堆的应用场景**：

1. **堆排序**：利用堆的性质进行排序，时间复杂度为 O(n log n)
2. **优先队列**：基于堆实现，用于任务调度、事件处理等
3. **Top K 问题**：找出最大或最小的 K 个元素，如找出数组中前 K 个最大元素
4. **中位数查找**：使用两个堆（最大堆和最小堆）维护中位数
5. **Dijkstra 算法**：用于寻找图中的最短路径
6. **Prim 算法**：用于寻找图中的最小生成树

**Java 中的堆实现**：

- **PriorityQueue**：基于最小堆实现的优先队列

- **Collections.max()**：使用堆找出集合中的最大值

- **Collections.min()**：使用堆找出集合中的最小值

## 11. 什么是图？图的遍历方式有哪些？

**图**是一种非线性数据结构，它由顶点（Vertex）和边（Edge）组成，用于表示元素之间的多对多关系。

**图的基本术语**：

- **顶点**：图中的元素，也称为节点

- **边**：连接两个顶点的线，分为有向边和无向边

- **有向图**：边有方向的图

- **无向图**：边没有方向的图

- **加权图**：边带有权重的图

- **邻接表**：图的一种表示方法，每个顶点维护一个相邻顶点的列表

- **邻接矩阵**：图的一种表示方法，使用二维数组表示顶点之间的连接关系

- **度**：顶点的边数，有向图中分为入度和出度

- **路径**：从一个顶点到另一个顶点的顶点序列

- **环**：起点和终点相同的路径

- **连通图**：任意两个顶点之间都有路径的无向图

- **强连通图**：任意两个顶点之间都有双向路径的有向图

**图的遍历方式**：

1. **深度优先搜索（DFS）**：

    - 从起始顶点开始，尽可能深地访问图中的顶点

    - 访问一个顶点后，递归访问其未访问的相邻顶点

    - 当所有相邻顶点都被访问后，回溯到上一个顶点

    - 可以使用栈或递归来实现

2. **广度优先搜索（BFS）**：

    - 从起始顶点开始，按层次访问图中的顶点

    - 访问一个顶点后，访问其所有未访问的相邻顶点

    - 然后依次访问这些相邻顶点的未访问相邻顶点

    - 使用队列来实现

**图遍历的实现**：

**邻接表表示的图**：

```java
// 图的邻接表表示
List<List<Integer>> graph = new ArrayList<>();

// 深度优先搜索
public void dfs(int start, boolean[] visited) {
    visited[start] = true;
    System.out.print(start + " ");
    for (int neighbor : graph.get(start)) {
        if (!visited[neighbor]) {
            dfs(neighbor, visited);
        }
    }
}

// 广度优先搜索
public void bfs(int start, boolean[] visited) {
    Queue<Integer> queue = new LinkedList<>();
    visited[start] = true;
    queue.offer(start);
    while (!queue.isEmpty()) {
        int curr = queue.poll();
        System.out.print(curr + " ");
        for (int neighbor : graph.get(curr)) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                queue.offer(neighbor);
            }
        }
    }
}
```

**图的应用场景**：

- 社交网络：顶点表示用户，边表示好友关系

- 路由算法：顶点表示路由器，边表示网络连接

- 地图导航：顶点表示地点，边表示道路

- 电路设计：顶点表示组件，边表示连接

- 搜索引擎：顶点表示网页，边表示超链接

## 12. 什么是深度优先搜索和广度优先搜索？

**深度优先搜索（Depth-First Search，DFS）**是一种图遍历算法，它从起始顶点开始，尽可能深地访问图中的顶点，直到无法继续深入，然后回溯到上一个顶点，继续探索其他路径。

**DFS 的特点**：

- 利用栈或递归实现

- 不保证找到最短路径

- 适合探索所有可能的路径

- 空间复杂度为 O(V)，其中 V 是顶点数

**DFS 的应用**：

- 拓扑排序

- 查找连通分量

- 检测环

- 解决迷宫问题

- 回溯算法（如八皇后问题、旅行商问题）

**广度优先搜索（Breadth-First Search，BFS）**是一种图遍历算法，它从起始顶点开始，按层次访问图中的顶点，先访问起始顶点的所有相邻顶点，然后依次访问这些相邻顶点的所有未访问相邻顶点。

**BFS 的特点**：

- 利用队列实现

- 保证找到最短路径（无权图）

- 适合寻找最短路径

- 空间复杂度为 O(V)，其中 V 是顶点数

**BFS 的应用**：

- 寻找最短路径

- 查找连通分量

- 检测环

- 层次遍历

- 解决迷宫问题

**DFS 和 BFS 的比较**：

| 特性           | DFS                                 | BFS                        |
| -------------- | ----------------------------------- | -------------------------- |
| **数据结构**   | 栈或递归                            | 队列                       |
| **访问顺序**   | 深度优先，先访问深层顶点            | 广度优先，先访问浅层顶点   |
| **最短路径**   | 不保证找到最短路径                  | 保证找到最短路径（无权图） |
| **空间复杂度** | O(V)                                | O(V)                       |
| **时间复杂度** | O(V + E)，其中 V 是顶点数，E 是边数 | O(V + E)                   |
| **应用场景**   | 探索所有可能路径、回溯算法          | 寻找最短路径、层次遍历     |

## 13. 什么是动态规划？动态规划的应用场景？

**动态规划（Dynamic Programming，DP）**是一种解决多阶段决策问题的优化算法，它通过将原问题分解为子问题，先求解子问题，然后从子问题的解得到原问题的解。

**动态规划的核心思想**：

- **最优子结构**：原问题的最优解包含子问题的最优解

- **重叠子问题**：子问题之间存在重叠，避免重复计算

- **状态转移方程**：描述问题状态之间的关系

- **记忆化存储**：将子问题的解存储起来，避免重复计算

**动态规划的基本步骤**：

1. 定义状态：确定问题的状态表示
2. 确定状态转移方程：描述状态之间的转移关系
3. 初始化状态：确定初始状态的取值
4. 计算顺序：确定计算状态的顺序（通常是自底向上）
5. 求解结果：从最终状态中得到原问题的解

**动态规划的应用场景**：

1. **最长公共子序列（LCS）**：

    - 问题：找出两个字符串的最长公共子序列

    - 状态：dp[i][j]表示字符串 A 的前 i 个字符和字符串 B 的前 j 个字符的最长公共子序列长度

    - 状态转移方程：如果 A[i] == B[j]，则 dp[i][j] = dp[i-1][j-1] + 1；否则，dp[i][j] = max(dp[i-1][j], dp[i][j-1])

2. **最长递增子序列（LIS）**：

    - 问题：找出数组中最长的递增子序列

    - 状态：dp[i]表示以第 i 个元素结尾的最长递增子序列长度

    - 状态转移方程：dp[i] = max(dp[j] + 1)，其中 j < i 且 nums[j] < nums[i]

3. **0-1 背包问题**：

    - 问题：给定一组物品，每个物品有重量和价值，在背包容量限制下，选择物品使得总价值最大

    - 状态：dp[i][j]表示前 i 个物品在容量 j 下的最大价值

    - 状态转移方程：如果第 i 个物品的重量大于 j，则 dp[i][j] = dp[i-1][j]；否则，dp[i][j] = max(dp[i-1][j], dp[i-1]j-w[i]] + v[i])

4. **斐波那契数列**：

    - 问题：计算第 n 个斐波那契数

    - 状态：dp[i]表示第 i 个斐波那契数

    - 状态转移方程：dp[i] = dp[i-1] + dp[i-2]

5. **爬楼梯问题**：

    - 问题：有 n 级台阶，每次可以爬 1 级或 2 级，有多少种不同的爬法

    - 状态：dp[i]表示爬到第 i 级台阶的不同爬法数

    - 状态转移方程：dp[i] = dp[i-1] + dp[i-2]

6. **编辑距离**：

    - 问题：计算将一个字符串转换为另一个字符串所需的最少操作次数（插入、删除、替换）

    - 状态：dp[i][j]表示将字符串 A 的前 i 个字符转换为字符串 B 的前 j 个字符所需的最少操作次数

    - 状态转移方程：如果 A[i] == B[j]，则 dp[i][j] = dp[i-1][j-1]；否则，dp[i][j] = min(dp[i-1][j-1] + 1, dp[i-1][j] + 1, dp[i][j-1] + 1)

**动态规划的优化**：

- **空间优化**：利用滚动数组将二维数组优化为一维数组

- **状态压缩**：将多个状态合并为一个状态

- **记忆化搜索**：将递归和记忆化结合，避免重复计算

## 14. 什么是贪心算法？贪心算法的应用场景？

**贪心算法**是一种在每一步选择中都采取当前状态下最优的选择，从而希望导致全局最优解的算法。贪心算法不考虑全局最优，只考虑局部最优，因此贪心算法不一定能得到全局最优解，但在某些问题中可以得到全局最优解。

**贪心算法的特点**：

- **局部最优选择**：在每一步选择中都采取当前状态下最优的选择

- **无后效性**：过去的选择不会影响未来的选择

- **贪心选择性质**：全局最优解可以通过局部最优解的选择得到

- **最优子结构**：问题的最优解包含子问题的最优解

**贪心算法的基本步骤**：

1. 确定问题的贪心策略
2. 证明贪心策略的正确性
3. 实现贪心算法

**贪心算法的应用场景**：

1. **活动选择问题**：

    - 问题：在一系列活动中选择最多的不重叠活动

    - 贪心策略：选择结束时间最早的活动，然后在剩余时间内选择结束时间最早的活动

2. **霍夫曼编码**：

    - 问题：为字符设计最优的前缀编码，使得总编码长度最小

    - 贪心策略：选择出现频率最低的两个字符合并，重复此过程直到只剩下一个字符

3. **分数背包问题**：

    - 问题：给定一组物品，每个物品有重量和价值，在背包容量限制下，选择物品使得总价值最大（物品可以分割）

    - 贪心策略：选择单位重量价值最高的物品，尽可能多地装入

4. **单源最短路径问题（Dijkstra 算法）**：

    - 问题：寻找从起始顶点到其他所有顶点的最短路径

    - 贪心策略：每次选择距离起始顶点最近的未访问顶点，更新其相邻顶点的距离

5. **最小生成树问题（Prim 算法和 Kruskal 算法）**：

    - 问题：寻找连接图中所有顶点的最小总权重的边集合

    - Prim 算法：从起始顶点开始，每次选择连接树和非树顶点的最小权重边

    - Kruskal 算法：按权重从小到大排序所有边，依次选择不形成环的边

6. **找零问题**：

    - 问题：用最少的硬币找零

    - 贪心策略：每次选择面值最大的硬币，尽可能多地使用

**贪心算法的局限性**：

- 贪心算法不一定能得到全局最优解

- 贪心策略的选择至关重要，不同的贪心策略可能得到不同的结果

- 需要证明贪心策略的正确性，否则可能得到错误的结果

## 15. 什么是哈希表？哈希表的冲突解决方法有哪些？

**哈希表**是一种根据键（Key）直接访问值（Value）的数据结构，它通过哈希函数将键映射到数组的索引，从而实现 O(1)时间复杂度的查找、插入和删除操作。

**哈希表的基本组成**：

- **数组**：存储值的容器

- **哈希函数**：将键映射到数组索引的函数

- **冲突解决机制**：处理不同键映射到同一索引的情况

**哈希函数的设计原则**：

- **确定性**：相同的键必须映射到相同的索引

- **均匀性**：键应均匀分布在数组中，减少冲突

- **高效性**：哈希函数的计算应快速

- **雪崩效应**：输入的微小变化应导致输出的较大变化

**常见的哈希函数**：

- 直接寻址法：直接使用键作为数组索引

- 除留余数法：h(key) = key % m，其中 m 是数组长度

- 数字分析法：选择数字的某些位作为哈希值

- 平方取中法：取键的平方的中间几位作为哈希值

- 折叠法：将键分割成若干部分，然后合并这些部分的哈希值

**哈希表的冲突解决方法**：

1. **开放定址法**：

    - 当发生冲突时，寻找数组中的下一个空位置来存储值

    - **线性探测**：h_i(key) = (h(key) + i) % m，其中 i 是探测次数

    - **二次探测**：h_i(key) = (h(key) + i²) % m

    - **双重哈希**：h_i(key) = (h1(key) + i \* h2(key)) % m，其中 h2(key)是另一个哈希函数

2. **链地址法**：

    - 每个数组元素存储一个链表，所有映射到同一索引的键值对都存储在该链表中

    - 当发生冲突时，将键值对添加到链表的末尾

    - Java 中的 HashMap 就是使用链地址法实现的，JDK 8+中当链表长度超过阈值（8）时，会转换为红黑树

3. **再哈希法**：

    - 当发生冲突时，使用另一个哈希函数重新计算哈希值

    - 直到找到空位置或确定元素不存在

4. **建立公共溢出区**：

    - 建立两个数组：基本数组和溢出数组

    - 所有冲突的元素都存储在溢出数组中

    - 查找时，先在基本数组中查找，找不到则在溢出数组中查找

**哈希表的性能**：

- **平均情况**：查找、插入和删除操作的时间复杂度为 O(1)

- **最坏情况**：当所有键都映射到同一索引时，时间复杂度为 O(n)

- **负载因子**：α = n / m，其中 n 是键值对数量，m 是数组长度

    - 负载因子越小，冲突越少，性能越好

    - 负载因子越大，冲突越多，性能越差

    - Java 中的 HashMap 默认负载因子为 0.75，当负载因子超过阈值时，会进行扩容

**哈希表的应用场景**：

- 实现字典、映射等数据结构

- 缓存系统

- 查找表

- 去重操作

- 计数统计

**Java 中的哈希表实现**：

- **HashMap**：基于链地址法实现的哈希表，线程不安全

- **Hashtable**：基于链地址法实现的哈希表，线程安全

- **LinkedHashMap**：基于链地址法实现的哈希表，保持插入顺序或访问顺序

- **ConcurrentHashMap**：基于分段锁实现的线程安全哈希表，JDK 8+中使用 CAS 和 synchronized 实现

## 16. 什么是字符串匹配算法？常见的字符串匹配算法有哪些？

**字符串匹配算法**是用于在主字符串中查找子字符串（模式串）的算法，它是字符串处理中的基础算法。

**常见的字符串匹配算法**：

1. **暴力匹配算法（Brute Force）**：

    - 从主字符串的每个位置开始，与模式串进行比较

    - 如果不匹配，从主字符串的下一个位置开始重新比较

    - 时间复杂度：O(n\*m)，其中 n 是主字符串长度，m 是模式串长度

    - 空间复杂度：O(1)

2. **KMP 算法（Knuth-Morris-Pratt）**：

    - 利用模式串的前缀和后缀的最长公共前缀后缀（LPS）数组，避免重复比较

    - 时间复杂度：O(n + m)

    - 空间复杂度：O(m)

    - 核心思想：当发生不匹配时，根据 LPS 数组跳过不必要的比较

3. **Boyer-Moore 算法**：

    - 从模式串的末尾开始比较，利用坏字符规则和好后缀规则跳过不必要的比较

    - 时间复杂度：最好情况下 O(n/m)，最坏情况下 O(n\*m)

    - 空间复杂度：O(k)，其中 k 是字符集大小

    - 实际应用中比 KMP 算法更快，被广泛应用于文本编辑器

4. **Rabin-Karp 算法**：

    - 使用哈希函数将字符串转换为哈希值，比较哈希值而不是字符串

    - 时间复杂度：平均情况下 O(n + m)，最坏情况下 O(n\*m)

    - 空间复杂度：O(1)

    - 核心思想：使用滚动哈希函数快速计算子串的哈希值

5. **Sunday 算法**：

    - 从模式串的末尾开始比较，当发生不匹配时，查看主字符串中模式串后一个字符的位置

    - 时间复杂度：平均情况下 O(n/m)，最坏情况下 O(n\*m)

    - 空间复杂度：O(k)，其中 k 是字符集大小

    - 实现简单，实际应用中性能较好

**字符串匹配算法的选择**：

- 小规模数据：暴力匹配算法

- 大规模数据：KMP 算法、Boyer-Moore 算法、Rabin-Karp 算法、Sunday 算法

- 实际应用：Boyer-Moore 算法和 Sunday 算法在实际应用中性能较好

**KMP 算法的 LPS 数组计算**：

```java
// 计算模式串的LPS数组
public int[] computeLPS(String pattern) {
    int m = pattern.length();
    int[] lps = new int[m];
    int len = 0; // 最长公共前缀后缀的长度
    int i = 1;

    while (i < m) {
        if (pattern.charAt(i) == pattern.charAt(len)) {
            len++;
            lps[i] = len;
            i++;
        } else {
            if (len != 0) {
                len = lps[len - 1];
            } else {
                lps[i] = 0;
                i++;
            }
        }
    }
    return lps;
}

// KMP算法实现
public int kmp(String text, String pattern) {
    int n = text.length();
    int m = pattern.length();
    if (m == 0) return 0;

    int[] lps = computeLPS(pattern);
    int i = 0; // text的索引
    int j = 0; // pattern的索引

    while (i < n) {
        if (text.charAt(i) == pattern.charAt(j)) {
            i++;
            j++;
        }

        if (j == m) {
            return i - j; // 找到匹配，返回起始位置
        } else if (i < n && text.charAt(i) != pattern.charAt(j)) {
            if (j != 0) {
                j = lps[j - 1];
            } else {
                i++;
            }
        }
    }
    return -1; // 未找到匹配
}
```

**字符串匹配算法的应用场景**：

- 文本编辑器中的查找和替换功能

- 搜索引擎中的关键词匹配

- 生物信息学中的 DNA 序列匹配

- 网络安全中的病毒特征码匹配

- 编译器中的词法分析
