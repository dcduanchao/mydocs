# AI 辅助面试回答提示词模板

## 适用场景

用于让 AI 基于当前仓库文档，快速生成适合面试现场的精简回答。目标不是写长篇解析，而是先给出 30 秒能说完的答案，再补高频坑和追问点。

## 通用提示词模板

```text
你是一个 Java 面试辅导助手。

请按下面规则回答我的问题：
1. 先在当前仓库里查找与问题最相关的文档内容，并指出具体文件路径与小节位置。
2. 基于仓库已有内容回答，不要脱离原文乱扩展；如果原文缺失，先说明“仓库暂无对应内容”，再做少量必要补充。
3. 如果具备文件修改能力，原文缺失时把补充内容同步写入最合适的文档位置；如果不能写文件，就输出可直接粘贴的 Markdown 片段。
4. 先输出“面试精简回答”：控制在 3 到 5 句话，口语化，适合 30 秒内回答完。
5. 再输出“展开补充”：只补最关键的 2 到 4 点。
6. 再输出“经典案例”：优先给代码案例；如果不适合写代码，再给真实面试场景。
7. 再输出“高频坑”：列出 3 到 5 个真实面试常问或常错点。
8. 如果这个问题容易被追问，再补“面试追问”，每个追问都要带一句简短参考回答。
9. 整体风格要简洁、直接、像候选人口述，不要写成教程。

我的问题是：{问题}
如果仓库无法读取，请提示我粘贴相关文档内容。
```

## 快捷口令

终极口令版：

```text
按 ai_interview_prompt 八股速刷：{你的问题}
```

稳妥版：

```text
按 ai_interview_prompt 八股速刷，先定位仓库答案：{你的问题}
```

示例：

```text
按 ai_interview_prompt 八股速刷，先定位仓库答案：Java 里 == 和 equals 的区别？
```

## 示例：`== vs equals` 

### 面试精简回答

`==` 比较的是值还是地址，要看类型。基本类型用 `==` 比较的是值，引用类型用 `==` 比较的是对象地址。`equals()` 主要给引用类型判断“内容是否相等”，但具体结果取决于类有没有重写 `equals()`。所以面试里一句话记住：基本类型看值，引用类型 `==` 看地址，内容比较优先看 `equals()`。

### 展开补充

- `Object` 默认的 `equals()` 本质上还是比较地址，所以如果类不重写，`equals()` 和 `==` 差不多。
- 像 `String`、`Integer` 这类常见类通常重写了 `equals()`，比较的是内容。
- 面试答题时最好主动补一句：比较对象内容时，不能只背 `equals()`，还要注意是否正确重写了 `hashCode()`。

### 经典案例（代码）

```java
String a = "abc";
String b = "abc";
System.out.println(a == b); // true，常量池复用

String x = new String("abc");
String y = new String("abc");
System.out.println(x == y); // false，不是同一个对象
System.out.println(x.equals(y)); // true，内容相同
```

### 高频坑

- `String a = "abc"; String b = "abc";`，`a == b` 通常是 `true`，因为字符串常量池复用对象。
- `String a = new String("abc"); String b = new String("abc");`，`a == b` 是 `false`，但 `a.equals(b)` 是 `true`。
- `Integer a = 127; Integer b = 127;`，`a == b` 是 `true`；超过缓存范围如 `128`，结果通常变成 `false`。
- 自定义类如果不重写 `equals()`，即使属性完全一样，`equals()` 也可能是 `false`。
- 用 `str.equals("abc")` 可能空指针，更稳妥的写法通常是 `"abc".equals(str)`。

### 面试追问

- `equals()` 和 `hashCode()` 为什么要一起重写？  
  因为哈希集合和哈希表会先用 `hashCode()` 定位，再用 `equals()` 判断是否相等，两者不一致会导致查找或去重异常。
- `String` 为什么会出现 `==` 有时为 `true`？  
  因为字符串常量池会复用相同字面量对象，所以两个字面量引用可能指向同一个对象。
- 包装类为什么会出现 `==` 时灵时不灵？  
  因为部分包装类有缓存范围，例如 `Integer` 默认缓存 `-128` 到 `127`，超出范围通常会创建新对象。

## 我的建议

这个提示词方向是对的，但要控制边界，不要让 AI “补充到天际”。

- 推荐保留四层输出：`面试精简回答`、`展开补充`、`经典案例`、`高频坑`，够用了。
- 不建议默认要求“大量案例”，否则答案会从面试口语稿变成教程。
- 最实用的做法是：先让 AI 从仓库定位答案，再只补 3 到 5 个高频坑，质量会比泛泛扩展更稳。
